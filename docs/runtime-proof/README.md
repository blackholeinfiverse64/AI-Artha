## Quick Start

```bash
# Start all services
docker compose up -d

# Verify backend health
curl http://localhost:5000/api/v1/health

# Run database proof
node backend/scripts/check-users.js

# Run replay proof
node backend/scripts/replay-provenance-proof.js

# Verify ledger chain
curl -H "Authorization: Bearer <TOKEN>" \
  http://localhost:5000/api/v1/ledger/verify-chain
```

## Runtime Mode Indicators

The frontend Financial Intelligence Dashboard displays the current runtime mode:

| Mode | Indicator | Meaning |
|------|-----------|---------|
| `BACKEND_CONNECTED` | Green banner | All APIs live, real DB data |
| `BACKEND_DEGRADED` | Amber banner | Some APIs unavailable, partial data |
| `BACKEND_UNAVAILABLE` | Red banner | Cannot reach port 5000 |
| `MOCK_MODE` | Purple banner | Development mock (should not appear in production) |

## Signal Pipeline Proof

```bash
# 1. Generate a compliance signal
curl -X GET -H "Authorization: Bearer <TOKEN>" \
  http://localhost:5000/api/v1/compliance/gst/gstr-1?period=$(date +%Y-%m)

# 2. List generated signals
curl -H "Authorization: Bearer <TOKEN>" \
  http://localhost:5000/api/v1/signals

# 3. Check pipeline for a signal
curl -H "Authorization: Bearer <TOKEN>" \
  http://localhost:5000/api/v1/signals/<SIGNAL_ID>/pipeline-check

# 4. Reconstruct trace
curl -H "Authorization: Bearer <TOKEN>" \
  http://localhost:5000/api/v1/signals/trace/<TRACE_ID>
```

## Hash Chain Verification

```bash
# Verify entire ledger hash chain
curl -H "Authorization: Bearer <TOKEN>" \
  http://localhost:5000/api/v1/ledger/verify-chain

# Expected response (intact chain):
# { "status": "ok", "data": { "verified": true, "entries_checked": N, "broken_at": null } }

# Expected response (tampered chain):
# { "status": "ok", "data": { "verified": false, "broken_at": "LE-<ID>", "entries_checked": N } }
```
